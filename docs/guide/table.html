Data(Card View)

The card view is another way of viewing data, expecially that which contains extra info like persons data, items for inventory etc. The front end of the card displays immediate info of the item and when a card is clicked, it slides up to display other details with buttons like, edit item, buy item, etc. We only want a maximum of two buttons to display at the back end. For inventory, we colud also have an input box to enter the number of items to purchase before clicking on the buy button.

DATA SWITCH TABEL-CARD View

In some cases, you might want to use this component to switch between list view to thumbnail view, likewise form the table view to the card view. This is very effective for data like persons and items. There will be a button at the header of the table which will toggle the views between table and card. In this case, you will declare the card view after the table view, the view will have a class "ad-show" to it. It would be better to use angular2 to demonstrate this since with javascript, we are able to stop the hidden view from loading, until its toggled to show. Using this in ordinary server-side language will load both views, which is a double data and memory consumption. it only hides one and shows the other with the same data presented in a different view.
